## 二、任务内容

### 2.1 文档结构

文档应包含以下章节：

1. **标题**：ADR-004: 选择 ATECC608A 作为安全芯片
2. **状态**：已接受（Accepted）
3. **背景**：
   - 项目需求（私钥存储、ES256 签名、密钥隔离）
   - 技术约束（私钥不可导出、安全等级）
   - 成本约束（安全芯片成本 < $5）
   - FIDO2 安全要求
   - 决策触发（需确定安全芯片以满足 FIDO2 安全要求）
4. **决策**：
   - 最终决策：ATECC608A-MAHDA-T
   - 决策理由（价格极低、开源项目验证、功能满足、CryptoAuthLib 库完善）
5. **考虑的选项**：
   - ATECC608A vs OPTIGA Trust M vs SE050 vs STSAFE-A100 对比分析
   - 详细技术参数对比表
   - 安全等级对比（EAL4+ vs EAL6+）
   - 成本对比
   - 功能对比
   - 开源项目支持对比
6. **技术规格**：
   - 安全等级：EAL4+
   - 密钥槽位：16 个
   - 支持算法：ECDSA (ES256)、ECDH (P-256)、SHA-256
   - 接口：I2C (高达 1 MHz)
   - 工作电压：2.0-5.5V
   - 工作电流：~5-10mA
   - 待机电流：< 1mA
   - 尺寸：8-pad SOIC
   - 温度范围：-40°C to +85°C
7. **成本分析**：
   - 单价：$1-2
   - 采购渠道建议
   - 批量采购优惠
   - 与其他安全芯片成本对比
8. **安全等级评估**：
   - EAL4+ 详细说明
   - 与 EAL5+/EAL6+ 对比
   - FIDO2 安全要求满足情况
   - 安全特性详细说明（防篡改、防侧信道攻击）
9. **功能评估**：
   - ES256 签名功能
   - ECDH 密钥协商功能
   - SHA-256 哈希功能
   - 密钥存储功能
   - 密钥生成功能
   - 防篡改检测功能
10. **风险评估**：
    - 安全等级风险（EAL4+ 是否足够）
    - 供应链风险
    - 开源库集成风险
    - 功能限制风险
11. **后果**：
    - 正面影响（成本极低、开源验证、功能满足）
    - 负面影响（安全等级较低、无 PIN 管理）
    - 后续决策依赖（LLD 设计、密钥管理方案）
12. **替代方案分析**：
    - OPTIGA Trust M 方案详细分析（高安全）
    - SE050 方案详细分析（企业级）
    - STSAFE-A100 方案详细分析（STM32 集成）
13. **实施计划**：
    - 硬件采购建议
    - CryptoAuthLib 集成方案
    - 密钥槽位分配方案
    - 验证里程碑
14. **备注**：
    - 参考资料
    - 更新记录
