## 一、主控芯片对比

### 1.1 候选芯片列表

| 芯片 | 架构 | 主频 | Flash | RAM | 价格（预估） | 开发难度 |
|------|------|------|-------|-----|--------------|----------|
| **ESP32** | Xtensa LX6 | 240 MHz | 4 MB | 520 KB | $3-5 | 低 |
| **ESP32-S3** | Xtensa LX7 | 240 MHz | 8 MB | 512 KB | $4-6 | 低 |
| **STM32L4** | ARM Cortex-M4 | 80 MHz | 1 MB | 128 KB | $5-8 | 中 |
| **STM32F4** | ARM Cortex-M4 | 168 MHz | 1 MB | 192 KB | $6-10 | 中 |
| **nRF52840** | ARM Cortex-M4F | 64 MHz | 1 MB | 256 KB | $5-7 | 中 |
| **nRF52833** | ARM Cortex-M4F | 64 MHz | 512 KB | 128 KB | $4-6 | 中 |
| **RP2040** | ARM Cortex-M0+ | 133 MHz | 外部 | 264 KB | $1-2 | 低 |
| **CH552** | 8051 | 24 MHz | 14 KB | 1.25 KB | $0.5-1 | 高 |
| **CH554** | 8051 | 24 MHz | 16 KB | 1.25 KB | $0.5-1 | 高 |

### 1.2 详细对比分析

#### ESP32 系列

**ESP32（原版）**
| 项目 | 说明 |
|------|------|
| **优点** | 价格低、社区活跃、WiFi/BLE 内置、开发工具丰富 |
| **缺点** | 功耗较高、无硬件安全特性、RAM 较小 |
| **适用性** | 适合 BLE 版认证器，需外接安全芯片 |
| **备注** | ESP-IDF 和 Arduino 支持完善 |

**ESP32-S3（新版）**
| 项目 | 说明 |
|------|------|
| **优点** | 更大 Flash、AI 加速、USB OTG 内置、支持 secure boot |
| **缺点** | 价格略高、功耗仍较高 |
| **适用性** | 推荐！USB 内置，减少外接芯片，支持安全启动 |
| **备注** | 适合 DIY FIDO2 项目 |

#### STM32 系列

**STM32L4（低功耗）**
| 项目 | 说明 |
|------|------|
| **优点** | 超低功耗、硬件安全特性（RDP、PCROP）、丰富的外设 |
| **缺点** | 价格较高、开发工具复杂（ST-Link）、无无线功能 |
| **适用性** | 适合 USB/NFC 版认证器，功耗敏感场景 |
| **备注** | 需外接安全芯片或使用 STM32L4 内置安全特性 |

**STM32F4（高性能）**
| 项目 | 说明 |
|------|------|
| **优点** | 性能强、USB OTG 内置、丰富的开发资源 |
| **缺点** | 功耗较高、价格高、无低功耗模式 |
| **适用性** | 适合 USB 版认证器，性能要求高场景 |
| **备注** | CubeMX 开发工具完善 |

#### nRF52 系列

**nRF52840**
| 项目 | 说明 |
|------|------|
| **优点** | BLE 5.0 内置、USB 内置、低功耗、硬件安全特性 |
| **缺点** | 价格较高、开发工具复杂、社区资源较少 |
| **适用性** | 推荐！BLE + USB 双模认证器 |
| **备注** | Nordic SDK 完善，适合 BLE 认证器 |

**nRF52833**
| 项目 | 说明 |
|------|------|
| **优点** | BLE 5.2 内置、价格低于 nRF52840、低功耗 |
| **缺点** | Flash 较小、无 USB |
| **适用性** | 适合 BLE 版认证器，需外接 USB 芯片 |
| **备注** | 适合纯 BLE 认证器 |

#### RP2040（Raspberry Pi Pico）

| 项目 | 说明 |
|------|------|
| **优点** | 价格极低（$1）、双核、USB 内置、社区活跃、C/C++/MicroPython 支持 |
| **缺点** | 无硬件安全特性、无无线功能、需要外部 Flash |
| **适用性** | 适合低成本 USB 认证器，需外接安全芯片 |
| **备注** | 开发门槛最低，适合初学者 |

#### CH55x 系列（国产低成本）

| 项目 | 说明 |
|------|------|
| **优点** | 价格极低、USB 内置、国产芯片 |
| **缺点** | 8051 架构、RAM 极小、开发困难、社区资源少 |
| **适用性** | 仅适合极低成本方案，功能受限 |
| **备注** | 不推荐用于 FIDO2 认证器 |

### 1.3 主控芯片推荐方案

| 排名 | 方案 | 适用场景 | 价格区间 |
|------|------|----------|----------|
| **推荐 1** | **ESP32-S3** | USB + BLE 认证器、中等成本、开发便捷 | $4-6 |
| **推荐 2** | **nRF52840** | BLE + USB 认证器、低功耗、专业方案 | $5-7 |
| **推荐 3** | **RP2040** | USB 认证器、极低成本、初学者友好 | $1-2 |
| **推荐 4** | **STM32L4** | USB/NFC 认证器、超低功耗、安全需求高 | $5-8 |

| 项目 | 价格 |

### 7.3 选型理由

1. **ESP32-S3**：USB OTG 内置减少外接芯片，BLE 内置支持无线认证，社区资源丰富，开发门槛低
2. **FPM383C**：价格适中，内置指纹算法减少开发工作量，UART 接口简单，小体积适合便携设备
3. **ATECC608A**：价格极低，开源项目（SoloKeys、OpenSK）验证可用，满足 ES256 签名要求
4. **USB + BLE**：双模支持增加使用场景，USB 为主、BLE 为辅
