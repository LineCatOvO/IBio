## 一、概述

### 1.1 IBio 项目简介

IBio 是一个开源的 FIDO2 安全密钥项目，旨在通过生物识别（指纹）验证增强身份认证的安全性。项目采用 DIY 设计理念，目标是实现低成本、高性能的 FIDO2 认证器。

**项目核心特性**：
- 符合 FIDO2/CTAP2.0/2.1 标准
- 指纹生物识别验证
- USB + BLE 双模通信
- 低成本方案（目标 < $31）
- 完全开源设计

### 1.2 硬件选型报告目的

本报告整合 IBio 项目所有硬件选型决策（ADR）和技术规格文档，为硬件设计、固件开发和系统集成提供完整的技术参考。

### 1.3 文档范围

本报告涵盖以下硬件组件的选型决策和技术规格：
- 主控芯片（MCU）
- 指纹传感器
- 安全芯片
- 通信模块
- 电源管理
- 存储方案

## 二、项目硬件需求总结

### 2.1 FIDO2 认证器硬件需求

根据 FIDO2/CTAP2 标准，认证器需满足以下核心硬件需求：

| 需求类别 | 具体需求 | 验收标准 |
|----------|----------|----------|
| **私钥存储** | 私钥不可导出，安全存储 | 私钥永不离开安全区域 |
| **签名功能** | ES256（ECDSA P-256 + SHA-256） | 签名时间 < 100ms |
| **用户验证** | 指纹验证或 PIN 验证 | FAR < 0.001%, FRR < 3% |
| **通信协议** | USB HID 和/或 BLE GATT | CTAP over HID/BLE |

### 2.2 FIDO2 核心要求

| FIDO2 要求 | 具体规格 | 硬件影响 |
|------------|----------|----------|
| **私钥不可导出** | 私钥必须存储在认证器内，不可导出 | 需要安全芯片或加密存储 |
| **ES256 签名** | ECDSA with SHA-256，曲线 P-256 | 需要支持 ECC 签名的硬件或软件实现 |
| **用户验证（UV）** | 指纹、PIN 或其他 | 需要指纹传感器或 PIN 输入接口 |
| **Credential ID** | 凭证标识存储 | 需要非易失性存储 |
| **签名计数器** | 防重放计数器 | 需要单调计数器或等效机制 |
| **attestation** | 设备认证签名 | 需要设备私钥和证书 |

### 2.3 性能要求

| 性能指标 | 目标值 | 说明 |
|----------|--------|------|
| **认证时间** | < 2 秒 | 从用户触摸到签名完成 |
| **签名时间** | < 100 ms | ES256 签名操作 |
| **指纹采集时间** | < 500 ms | 指纹图像采集 |
| **指纹比对时间** | < 1 秒 | 1:1 或 1:N 比对 |
| **USB 响应** | < 30 秒 | CTAP 响应超时 |
| **BLE 响应** | < 30 秒 | CTAP 响应超时 |

### 2.4 生物识别需求

| 指标 | FIDO2 要求 | IBio 目标 |
|------|------------|-----------|
| **误识率（FAR）** | < 0.001% (1/100,000) | < 0.001% |
| **拒识率（FRR）** | < 3% | < 2% |
| **匹配时间** | < 2 秒 | < 1 秒 |
| **本地比对** | 必须 | 必须（模板不可导出） |
| **模板容量** | 每用户至少 5 个 | 最多 200 个 |

### 2.5 通信需求

| 通信模式 | 协议 | 说明 |
|----------|------|------|
| **USB HID** | CTAP over HID | 桌面认证（Windows/Mac/Linux） |
| **BLE GATT** | CTAP over BLE | 移动认证（iOS/Android） |
| **双模支持** | USB + BLE | 同时支持，无缝切换 |

### 2.6 成本需求

| 成本项目 | 目标成本 | 说明 |
|----------|----------|------|
| **总硬件成本** | < $31 | 核心硬件 + PCB + 连接器 |
| **主控芯片** | $5-8 | MCU 模块 |
| **指纹传感器** | $10-15 | 指纹模块 |
| **安全芯片** | $1-2 | 加密芯片 |
| **PCB + 元件** | $5-8 | PCB 和其他元件 |

### 2.7 功耗需求

| 功耗状态 | 目标值 | 说明 |
|----------|--------|------|
| **待机功耗** | < 10 mA | 等待用户操作 |
| **工作功耗** | < 100 mA | 签名/指纹采集 |
| **峰值功耗** | < 150 mA | 所有模块同时工作 |
