## 一、概述

### 1.1 FPM383C 简介

FPM383C 是由 Fingerprint Cards（FPC）公司开发的电容式指纹传感器模块。该模块集成了指纹采集、特征提取、模板比对等功能，提供完整的指纹识别解决方案。模块采用 UART 接口通信，适合嵌入式系统集成。

**核心特性**：
- 电容式按压传感器，检测指纹脊谷电容差异
- 内置指纹算法，无需外部处理
- 模板存储安全，禁止导出
- 小体积设计，适合便携设备
- 低功耗设计，适合电池供电设备

### 1.2 核心特性汇总

| 特性项 | 参数 |
|--------|------|
| **传感器类型** | 电容式按压传感器 |
| **分辨率** | 192×192 像素（508 DPI） |
| **采集面积** | 10×10 mm |
| **模板容量** | 200 个指纹模板 |
| **误识率（FAR）** | < 0.001%（1/100,000） |
| **拒识率（FRR）** | < 2% |
| **匹配时间** | < 1 秒 |
| **接口类型** | UART |
| **工作电压** | 3.3V DC（2.8V-3.6V） |
| **工作电流** | 20-50mA（采集时） |
| **待机电流** | < 5mA |
| **工作温度** | -20°C ~ +60°C |

### 1.3 适用场景

| 应用场景 | 说明 |
|----------|------|
| **FIDO2 安全密钥** | 本地指纹验证，符合 FIDO2 生物识别认证标准 |
| **智能门锁** | 指纹开锁，本地比对安全 |
| **便携设备** | 小体积适合便携集成 |
| **嵌入式系统** | UART 接口简单集成 |
| **消费电子产品** | 低成本方案，适合量产 |

## 三、性能规格

### 3.1 识别精度

| 规格项 | 参数值 | FIDO2 要求 | 满足情况 |
|--------|--------|------------|----------|
| **误识率（FAR）** | < 0.001%（1/100,000） | < 0.001% | **满足** |
| **拒识率（FRR）** | < 2% | < 3% | **满足** |
| **安全等级** | 1-5 级可配置 | - | 可调整 |

**说明**：
- **FAR（False Acceptance Rate）**：错误接受率，非注册用户被错误识别为注册用户的概率
- **FRR（False Rejection Rate）**：错误拒绝率，注册用户被错误拒绝识别的概率
- FIDO2 标准要求 FAR < 0.001%、FRR < 3%
- FPM383C 实际性能优于 FIDO2 要求

### 3.2 识别速度

| 操作类型 | 厂商声明 | 实测数据 | 用户体验 |
|----------|----------|----------|----------|

| **1:N 搜索** | < 1 秒 | 约 0.8 秒（200 模板） | 快速 |
| **指纹注册** | < 3 秒（多次采集） | 约 2.5 秒（3 次） | 良好 |
| **单次采集** | < 0.5 秒 | 约 0.3 秒 | 快速 |
| **特征提取** | 内置处理 | 与采集同步 | 自动 |

## 九、通信协议

### 9.1 协议概述

FPM383C 采用自定义 UART 协议进行通信。所有命令和响应均采用固定格式的数据包，包含包头、地址、命令码、数据和校验字段。

### 9.2 数据包格式

| 字段 | 字节数 | 值/说明 |
|------|--------|----------|
| **包头（Header）** | 2 bytes | 固定值 0xEF 0x01 |
| **地址（Address）** | 4 bytes | 模块地址，默认 0xFFFFFFFF |
| **包标识（Package Identifier）** | 1 byte | 命令包=0x01，数据包=0x02，响应包=0x07 |
| **包长度（Package Length）** | 2 bytes | 数据区长度（含校验） |
| **命令码（Command Code）** | 1 byte | 具体命令代码 |
| **数据（Data）** | N bytes | 命令参数或响应数据 |
| **校验（Checksum）** | 2 bytes | 包标识+包长度+命令码+数据的校验和 |

### 9.3 数据包结构示意

```
发送命令包格式（MCU → FPM383C）：
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│ Header │ Address│ PID    │ Length │ CMD    │ Data   │ Checksum│
│ 2 bytes│ 4 bytes│ 1 byte │ 2 bytes│ 1 byte │ N bytes│ 2 bytes │
│ EF 01  │ FF FF FF FF │ 01 │ N+2   │ XX    │ ...    │ XX XX  │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┘

响应包格式（FPM383C → MCU）：
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│ Header │ Address│ PID    │ Length │ Confirm│ Data   │ Checksum│
│ 2 bytes│ 4 bytes│ 1 byte │ 2 bytes│ 1 byte │ N bytes│ 2 bytes │
│ EF 01  │ FF FF FF FF │ 07 │ N+2   │ XX    │ ...    │ XX XX  │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┘
```

### 9.4 包标识定义

| 包标识值 | 名称 | 说明 |
|----------|------|------|
| **0x01** | 命令包 | MCU 发送的命令 |
| **0x02** | 数据包 | 数据传输（如图像数据） |
| **0x07** | 响应包 | 模块返回的响应 |
| **0x08** | 结束包 | 数据传输结束 |

|----------|------|------|
| **0x00** | 成功 | 命令执行成功 |

| **0x02** | 无手指 | 未检测到手指 |
| **0x03** | 采集失败 | 指纹采集失败 |
| **0x04** | 模板不完整 | 注册模板未完成 |
| **0x05** | 模板存储失败 | 模板存储失败 |
| **0x06** | 模板不匹配 | 指纹比对不匹配 |
| **0x07** | 模板 ID 无效 | 指定的模板 ID 不存在 |
| **0x08** | 模板库满 | 模板存储空间已满 |
| **0x09** | 模板库空 | 模板库无模板 |
| **0x0A** | 参数错误 | 命令参数无效 |

### 9.7 常用命令详细说明

#### 采集指令（0x01）

**命令格式**：
```
发送：EF01 FFFFFFFF 01 03 01 00 05
响应：EF01 FFFFFFFF 07 03 [确认码] [校验]
```

**功能**：采集指纹图像并存入模块内部缓冲区。

**参数**：无

**响应确认码**：
- 0x00：采集成功
- 0x02：无手指
- 0x03：采集失败

---

#### 比对指令（0x03）

**命令格式**：
```
发送：EF01 FFFFFFFF 01 05 03 [缓冲区号] [模板ID] [校验]
响应：EF01 FFFFFFFF 07 05 [确认码] [匹配分数] [校验]
```

**功能**：将缓冲区指纹与指定模板 ID 进行 1:1 比对。

**参数**：
- 缓冲区号：通常为 1 或 2
- 模板 ID：要比对的模板编号（0-199）

**响应数据**：
- 确认码：0x00（匹配），0x06（不匹配）
- 匹配分数：0-100 或 0-255

---

#### 搜索指令（0x04）

**命令格式**：
```
发送：EF01 FFFFFFFF 01 08 04 [缓冲区号] [起始ID] [模板数量] [校验]
响应：EF01 FFFFFFFF 07 07 [确认码] [匹配ID] [匹配分数] [校验]
```

**功能**：在模板库中搜索匹配指纹（1:N 搜索）。

**参数**：
- 缓冲区号：通常为 1 或 2
- 起始 ID：搜索起始模板 ID（通常 0 或 1）
- 模板数量：搜索范围（通常 200）

**响应数据**：
- 确认码：0x00（找到），0x09（未找到）
- 匹配 ID：找到的模板编号
- 匹配分数：匹配分数值

---

#### 注册流程（0x05/0x06/0x07）

**步骤一：注册开始（0x05）**
```
发送：EF01 FFFFFFFF 01 04 05 [缓冲区号] [校验]
响应：EF01 FFFFFFFF 07 03 [确认码] [校验]
```

**步骤二：续采集（0x06）**
```
发送：EF01 FFFFFFFF 01 04 06 [缓冲区号] [校验]
响应：EF01 FFFFFFFF 07 03 [确认码] [校验]
```

**步骤三：注册完成（0x07）**
```
发送：EF01 FFFFFFFF 01 06 07 [缓冲区号] [模板ID] [校验]
响应：EF01 FFFFFFFF 07 03 [确认码] [校验]
```

**功能**：通过多次采集完成指纹注册。

**建议采集次数**：3-5 次

---

#### 删除模板（0x0C）

**命令格式**：
```
发送：EF01 FFFFFFFF 01 06 0C [模板ID] [删除数量] [校验]
响应：EF01 FFFFFFFF 07 03 [确认码] [校验]
```

**功能**：删除指定模板。

**参数**：
- 模板 ID：起始删除的模板编号
- 删除数量：要删除的模板数量（1 表示删除单个）

---

#### 查询模板数量（0x1D）

**命令格式**：
```
发送：EF01 FFFFFFFF 01 03 1D 00 1E
响应：EF01 FFFFFFFF 07 05 [确认码] [模板数量] [校验]
```

**功能**：查询当前已存储的模板数量。

---

### 9.8 校验计算方法

校验和计算方式：将包标识、包长度、命令码、数据依次相加，结果取 16 位值。

**计算公式**：
```
Checksum = (PID + Length_H + Length_L + CMD + Data[0] + ... + Data[N]) & 0xFFFF
```

## 十一、应用建议

### 11.1 电路设计建议

#### 电源设计

| 设计项 | 建议 | 说明 |
|--------|------|------|
| **稳压芯片选择** | LDO（如 AMS1117-3.3） | 提供 3.3V 稳定输出 |
| **电源容量** | ≥ 100mA | 考虑峰值电流 |
| **滤波电容** | 10-100μF 电解电容 | 稳定电源电压 |
| **去耦电容** | 0.1μF 陶瓷电容 | 高频滤波 |
| **电源指示** | LED + 电阻 | 可视化电源状态 |

#### UART 接口设计

| 设计项 | 建议 | 说明 |
|--------|------|------|
| **电平匹配** | 确保均为 3.3V 电平 | 避免电平转换 |
| **TX/RX 连接** | MCU TX → 模块 RX | 交叉连接 |
| **波特率配置** | 推荐使用 57600 bps | 平衡速度与稳定性 |
| **接收缓冲** | MCU 需缓冲区 | 接收响应数据 |
| **超时处理** | 设置超时时间（如 2 秒） | 防止通信阻塞 |

#### GPIO 控制设计

| 设计项 | 建议 | 说明 |
|--------|------|------|
| **EN 引脚控制** | GPIO 输出控制 | 低功耗休眠控制 |
| **TOUCH 引脚** | GPIO 输入检测 | 触摸中断检测 |
| **上拉/下拉** | 根据需求配置 | 稳定信号 |
| **保护电阻** | 可选串联电阻 | 限流保护 |

### 11.2 UART 配置建议

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **波特率** | 57600 bps | 推荐使用，兼顾速度和稳定性 |
| **数据格式** | 8N1 | 8 数据位、无校验、1 停止位 |
| **超时时间** | 2000 ms | 响应超时设置 |
| **重试次数** | 3 次 | 通信失败重试 |
| **接收缓冲** | 256 bytes | 接收数据缓冲区 |

### 11.3 指纹采集建议

| 建议 | 说明 |
|------|------|
| **采集前清洁** | 清洁传感器表面和手指 |
| **干燥手指处理** | 干燥手指多次采集，提高灵敏度 |
| **湿润手指处理** | 清洁手指，降低灵敏度 |
| **采集姿势** | 手指自然平放，覆盖传感器区域 |
| **采集时间** | 稳定按压约 0.5 秒 |
| **质量评估** | 根据确认码判断采集质量 |

### 11.4 模板管理建议

| 建议 | 说明 |
|------|------|
| **注册次数** | 建议 3-5 次采集，提高模板质量 |
| **模板 ID 管理** | 建议模板 ID 映射表，跟踪用户关联 |
| **模板备份** | 模块不支持导出，外部记录 ID 映射 |
| **模板更新** | 定期更新模板，保持识别准确性 |
| **模板清理** | 定期清理无效模板，释放存储空间 |

### 11.5 性能优化建议

| 优化项 | 建议方法 |
|--------|----------|
| **降低 FRR** | 多次采集注册，提高模板质量 |
| **提升速度** | 使用 57600 或 115200 波特率 |
| **降低功耗** | 使用 EN 引脚控制休眠 |
| **改善用户体验** | 添加 LED 状态指示 |
| **提高稳定性** | 添加滤波电容，稳定电源 |

### 11.6 FIDO2 集成建议

| 集成项 | 建议方法 |
|--------|----------|
| **用户验证流程** | CTAP GetAssertion → 指纹采集 → 比对 → 返回结果 |
| **UV 标志** | 指纹验证成功返回 UV=1（用户验证成功） |
| **失败处理** | 连续失败 3-5 次，返回 UV Blocked |
| **模板关联** | 指纹模板与 CTAP 凭证独立管理 |
| **Reset 处理** | CTAP Reset 命令触发删除所有模板 |

### 11.7 PCB 设计建议

| 设计项 | 建议 |
|--------|------|
| **模块布局** | 传感器表面朝上，便于用户按压 |
| **排针间距** | 2.54mm 间距，标准排针 |
| **电源走线** | 独立电源走线，足够宽度 |
| **UART 走线** | 避免与高速信号交叉 |
| **ESD 保护** | 传感器表面增加 ESD 保护 |
| **外壳设计** | 需外壳保护传感器，露出采集区域 |
