## 一、项目概述

### 1.1 项目背景

#### 密码认证安全危机

当前网络安全面临严峻挑战，传统密码认证存在以下核心问题：

| 问题类型 | 具体表现 | 影响范围 |
|----------|----------|----------|
| **密码泄露** | 大规模数据库泄露事件频发 | 数亿用户账户受影响 |
| **钓鱼攻击** | 伪装登录页面窃取密码 | 企业和个人均受威胁 |
| **弱密码** | 用户习惯使用简单密码 | 账户易被暴力破解 |
| **密码复用** | 同一密码多平台使用 | 单点泄露影响多平台 |
| **管理负担** | 需记忆大量密码 | 用户体验差 |

根据 Verizon 数据泄露调查报告，超过 80% 的数据泄露事件涉及弱密码或被盗密码。

#### FIDO2 标准发展趋势

FIDO（Fast IDentity Online）联盟提出的 FIDO2 标准正在重塑身份认证行业：

| 标准组件 | 内容 | 状态 |
|----------|------|------|
| **WebAuthn** | W3C 制定的浏览器 API 标准 | 已成为浏览器标准 |
| **CTAP** | 客户端与认证器通信协议 | CTAP 2.0/2.1 已发布 |
| **无密码认证** | 消除密码依赖 | 主流平台已支持 |

主流平台对 FIDO2 的支持情况：

| 平台 | FIDO2 支持状态 | 说明 |
|------|----------------|------|
| **Windows 10/11** | 完全支持 | Windows Hello 集成 |
| **macOS** | 完全支持 | Safari 浏览器集成 |
| **iOS 14+** | 完全支持 | Face ID/Touch ID 集成 |
| **Android 9+** | 完全支持 | 生物识别集成 |
| **Chrome** | 完全支持 | WebAuthn API |
| **Firefox** | 完全支持 | WebAuthn API |
| **Edge** | 完全支持 | WebAuthn API |

#### 市场需求分析

**个人用户需求**：
- 消除密码管理负担
- 提高账户安全性
- 获得便捷的登录体验
- 降低密码泄露风险

**企业用户需求**：
- 增强员工账户安全
- 减少密码管理成本
- 符合安全合规要求
- 防止钓鱼攻击入侵

**开发者/DIY 需求**：
- 学习 FIDO2 技术实现
- 自制安全密钥设备
- 定制化硬件方案
- 低成本实验平台

| **ESP32-S3 供应短缺** | 低 | 高 | 多供应商采购、提前采购 |
| **FPM383C 供应短缺** | 中 | 高 | 多供应商、替代传感器（GT521F32） |
| **ATECC608A 供应短缺** | 低 | 中 | 多供应商（DigiKey/Mouser） |
| **PCB 制造周期** | 低 | 低 | 多 PCB 制造商备选 |

供应链风险分析：

- ESP32-S3：乐鑫官方供应稳定
- FPM383C：国内供应商较多，替代方案可用
- ATECC608A：Microchip 官方渠道稳定

### 5.4 合规风险

| 风险项 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **无 FIDO 官方认证** | 确定 | 中 | 明确 DIY 项目定位、用户知晓 |
| **功能合规性** | 中 | 高 | 参考 FIDO2 规范、官方测试工具 |
| **安全合规性** | 中 | 高 | 参考 FIDO2 安全要求、安全测试 |

合规风险分析：

- DIY 项目明确不追求 FIDO Alliance 官方认证
- 通过参考 FIDO2 规范实现功能合规
- 用户知晓项目定位和使用场景

### 5.5 风险矩阵汇总

| 风险等级 | 风险数量 | 关键风险项 |
|----------|----------|------------|
| **高** | 2 | FPM383C 供应、功能合规性 |
| **中** | 6 | BLE 稳定性、市场接受度等 |
| **低** | 4 | USB HID、FPM383C SDK 等 |
