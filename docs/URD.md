## 一、用户概述

### 1.1 目标用户群体

| 用户群体 | 特征描述 | 核心需求 |
|----------|----------|----------|
| **个人用户** | 关注账户安全、厌倦密码管理、追求便捷登录体验 | 低成本、易使用、安全可靠 |

| **DIY/开发者用户** | 技术爱好者、嵌入式开发者、安全研究者 | 开源可定制、学习价值、可复现 |

## 二、系统能力（用户视角）

### 2.1 核心认证能力

从用户视角，IBio 提供以下核心认证能力：

| 能力 | 用户视角描述 | 用户价值 |
|------|--------------|----------|
| **无密码登录** | 用指纹代替密码登录网站，无需记忆和输入密码 | 消除密码管理负担 |
| **一键注册** | 在新网站注册时，触摸设备即可创建账户凭证 | 注册流程简化 |
| **指纹验证** | 用手指触摸设备完成身份验证，1 秒内完成 | 快速便捷的用户体验 |
| **多网站支持** | 一个设备可用于多个网站（邮箱、社交、银行等） | 减少认证设备数量 |

### 2.2 通信连接能力

| 能力 | 用户视角描述 | 用户价值 |
|------|--------------|----------|
| **USB 直连** | 将设备插入电脑 USB-C 接口，自动识别使用 | 桌面端便捷使用 |
| **无线连接** | 通过蓝牙与手机连接，无需物理接触 | 移动端便捷使用 |
| **双模切换** | USB 和蓝牙自动切换，适应不同设备 | 多设备场景无缝体验 |

### 2.3 安全保护能力

| 能力 | 用户视角描述 | 用户价值 |
|------|--------------|----------|
| **防钓鱼** | 设备自动验证网站真伪，不在钓鱼网站工作 | 防止钓鱼攻击窃取凭证 |
| **防密码泄露** | 无密码存储，密码泄露事件不影响安全 | 消除密码泄露风险 |
| **私钥不可导出** | 设备内密钥永远不离开设备，无法被窃取 | 防止密钥被盗 |
| **指纹本地验证** | 指纹数据只在设备内比对，不上传到网站 | 保护隐私和生物信息 |

### 2.4 凭证管理能力

| 能力 | 用户视角描述 | 用户价值 |
|------|--------------|----------|
| **凭证自动存储** | 注册时设备自动存储凭证，无需手动管理 | 简化凭证管理 |
| **凭证安全删除** | 可删除不再使用的网站凭证 | 凭证灵活管理 |
| **设备重置** | 可完全重置设备，清除所有凭证和指纹 | 设备转让或清理场景 |

## 三、使用场景

### 3.1 注册场景

#### 场景描述

用户首次在某网站创建账户，使用 IBio 设备代替传统密码注册流程。

#### 用户操作流程

```
步骤 1：用户访问网站注册页面
步骤 2：网站提示"使用安全密钥注册"
步骤 3：用户将 IBio 设备插入电脑 USB-C 接口
步骤 4：网站检测到设备，提示"请触摸设备完成注册"
步骤 5：用户用手指触摸 IBio 指纹传感器
步骤 6：IBio 采集指纹，验证用户身份
步骤 7：验证成功后，IBio 生成凭证并存储
步骤 8：网站显示"注册成功"
步骤 9：用户完成注册，无需设置密码
```

#### 系统响应（用户可见）

| 用户操作 | 系统响应 | 用户可见反馈 |
|----------|----------|--------------|
| 插入设备 | 设备被识别 | 网站显示"设备已连接" |
| 触摸传感器 | 设备 LED 亮起 | LED 绿灯闪烁 |
| 指纹验证成功 | 凭证生成完成 | LED 绿灯常亮，网站显示"注册成功" |
| 指纹验证失败 | 拒绝注册 | LED 红灯闪烁，网站显示"验证失败，请重试" |

#### 场景时长

- **预期时长**：约 5-10 秒
- **指纹验证时间**：< 1 秒
- **凭证生成时间**：< 3 秒

---

### 3.2 认证（登录）场景

#### 场景描述

用户使用 IBio 设备登录已注册的网站，代替传统密码登录流程。

#### 用户操作流程

```
步骤 1：用户访问网站登录页面
步骤 2：网站提示"使用安全密钥登录"
步骤 3：用户将 IBio 设备插入电脑 USB-C 接口
        或通过蓝牙连接手机
步骤 4：网站检测到设备，提示"请触摸设备完成登录"
步骤 5：用户用手指触摸 IBio 指纹传感器
步骤 6：IBio 采集指纹，验证用户身份
步骤 7：验证成功后，IBico 使用私钥签名挑战
步骤 8：网站验证签名，确认用户身份
步骤 9：网站显示"登录成功"
步骤 10：用户进入网站账户
```

#### 系统响应（用户可见）

| 用户操作 | 系统响应 | 用户可见反馈 |
|----------|----------|--------------|
| 插入/连接设备 | 设备被识别 | 网站显示"设备已连接" |
| 触摸传感器 | 设备 LED 亮起 | LED 绿灯闪烁 |
| 指纹验证成功 | 签名完成 | LED 绿灯常亮，网站显示"登录成功" |
| 指纹验证失败 | 拒绝登录 | LED 红灯闪烁，网站显示"验证失败，请重试" |

#### 场景时长

- **预期时长**：约 2-5 秒
- **指纹验证时间**：< 1 秒
- **签名时间**：< 100 ms

### 3.3 指纹管理场景

#### 3.3.1 指纹注册场景

##### 场景描述

用户首次使用设备，需要注册指纹以启用指纹验证功能。

##### 用户操作流程

```
步骤 1：用户将 IBio 设备插入电脑
步骤 2：用户打开 IBio 配置工具（可选）
步骤 3：用户选择"注册指纹"
步骤 4：设备提示"请按压指纹传感器"
步骤 5：用户用手指按压传感器（多次）
步骤 6：设备采集指纹图像并生成模板
步骤 7：设备提示"指纹注册成功"
步骤 8：用户可为指纹命名（如"左手食指"）
```

##### 系统响应（用户可见）

| 用户操作 | 系统响应 | 用户可见反馈 |
|----------|----------|--------------|
| 按压传感器 | 指纹采集开始 | LED 绿灯闪烁 |
| 采集成功 | 模板生成进度 | 配置工具显示进度条 |
| 注册完成 | 模板存储成功 | LED 绿灯常亮，工具显示"注册成功" |
| 注册失败 | 指纹质量不足 | LED 红灯闪烁，工具提示"请重新按压" |

##### 注册要求

- **按压次数**：约 3-5 次（生成高质量模板）
- **每次按压时长**：约 1-2 秒
- **总时长**：约 15-30 秒

## 四、用户交互设计

### 4.1 物理交互

#### 4.1.1 触摸交互

| 交互方式 | 触发条件 | 用户感受 | 系统响应 |
|----------|----------|----------|----------|
| **轻触传感器** | 手指轻触指纹传感器表面 | 传感器表面平滑、触感舒适 | 触摸事件触发 |
| **按压传感器** | 手指适度按压传感器 | 按压反馈明确、力度适中 | 指纹采集开始 |
| **保持按压** | 保持按压约 1 秒 | 等待时间短、不焦虑 | 指纹图像采集完成 |

**触摸交互设计要求**：

| 要求 | 说明 |
|------|------|
| **触摸区域大小** | 至少 10×10 mm，便于手指定位 |
| **触摸反馈** | 触摸时 LED 立即亮起，确认触摸有效 |
| **等待时间** | 指纹采集 < 500 ms，用户无需长时间等待 |
| **误触容忍** | 短暂触碰不触发，持续按压才触发 |

#### 4.1.2 LED 指示

| LED 状态 | LED 颜色 | 含义 | 用户理解 |
|----------|----------|------|----------|
| **熄灭** | 无 | 设备空闲/待机 | 设备未在工作 |
| **绿灯闪烁** | 绿色 | 设备等待用户触摸 | "请触摸设备" |
| **绿灯常亮** | 绿色 | 操作成功完成 | "操作成功" |
| **红灯闪烁** | 红色 | 操作失败/错误 | "操作失败，请重试" |
| **红灯常亮** | 红色 | 设备错误/锁定 | "设备异常" |
| **交替闪烁** | 绿→红交替 | 设备配对/连接中 | "设备正在连接" |

**LED 指示设计要求**：

| 要求 | 说明 |
|------|------|
| **LED 可见性** | LED 在设备顶部，用户可见 |
| **LED 颜色区分** | 绿色（成功）、红色（失败）清晰区分 |
| **LED 闪烁频率** | 闪烁频率约 1 Hz，易于辨识 |
| **LED 亮度** | 亮度适中，室内可见但不刺眼 |

### 4.2 认证器状态（用户视角）

用户可感知的认证器状态：

| 状态 | 用户感知 | 状态描述 | 用户可操作 |
|------|----------|----------|------------|
| **待机状态** | LED 熄灭 | 设备已连接但未工作 | 无需操作，等待网站调用 |
| **等待触摸** | LED 绿灯闪烁 | 网站请求认证，等待用户触摸 | 用户触摸传感器 |
| **采集状态** | LED 绿灯快速闪烁 | 正在采集指纹图像 | 保持按压 |
| **成功状态** | LED 绿灯常亮（约 2 秒） | 认证成功完成 | 无需操作，可拔出设备 |
| **失败状态** | LED 红灯闪烁（约 2 秒） | 认证失败 | 可重新触摸尝试 |
| **锁定状态** | LED 纯灯常亮 | 指纹多次失败，设备锁定 | 需等待解锁或使用 PIN |

### 7.2 术语表（用户视角）

| **钓鱼网站** | 伪装成真实网站的虚假网站，用于窃取信息 |
| **WebAuthn** | 网站支持安全密钥登录的技术标准 |
