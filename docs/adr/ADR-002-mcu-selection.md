# ADR-002: 选择 ESP32-S3 作为主控芯片

**状态**：已接受（Accepted）

**决策日期**：2026-04-06

**决策者**：IBio 项目团队

---

## 一、标题

选择 ESP32-S3-WROOM-1-N8R8 作为 IBio FIDO2 安全密钥的主控芯片

---

## 二、背景

### 2.1 项目需求

IBio 项目是一个 DIY FIDO2 安全密钥，主控芯片需要满足以下核心需求：

| 需求类别 | 具体需求 | 验收标准 |
|----------|----------|----------|
| **通信需求** | USB HID + BLE GATT 双模 | USB 和 BLE 同时支持 CTAP 协议 |
| **性能需求** | 足够运行 FIDO2 协议栈 | CTAP 响应时间 < 2 秒 |
| **存储需求** | 存储凭证和固件 | 至少 6 MB 可用存储空间 |
| **外设需求** | UART/I2C/SPI/GPIO | 连接指纹传感器、安全芯片 |
| **成本需求** | 主控成本适中 | 主控芯片成本 $5-8 |
| **开发需求** | 开发便捷、社区活跃 | 完善的 SDK 和开发工具 |

### 2.2 技术约束

| 约束条件 | 说明 |
|----------|------|
| **USB HID 约束** | 需实现 USB HID 设备，支持 CTAP over HID |
| **BLE GATT 约束** | 需实现 FIDO GATT 服务，支持 CTAP over BLE |
| **安全芯片集成** | 需通过 I2C 与 ATECC608A 安全芯片通信 |
| **指纹传感器集成** | 需通过 UART 与 FPM383C 指纹模块通信 |
| **功耗约束** | USB 供电时功耗 < 100mA，待机功耗 < 10mA |
| **尺寸约束** | 需适合便携设备，模块尺寸应紧凑 |

### 2.3 成本约束

项目目标总成本 < $31，主控芯片成本分配：

| 成本项 | 目标成本 |
|--------|----------|
| **主控芯片** | $5-8 |
| **指纹传感器** | $10-15 |
| **安全芯片** | $1-2 |
| **其他元件** | $5-8 |

### 2.4 决策触发

需要确定主控芯片型号以便：
1. 开始 PCB 硬件设计
2. 进行固件架构设计
3. 制定硬件采购计划
4. 建立开发环境

---

## 三、决策

### 3.1 最终决策

**选择 ESP32-S3-WROOM-1-N8R8 作为主控芯片**

| 项目 | 具体型号 |
|------|----------|
| **芯片型号** | ESP32-S3 |
| **模块型号** | ESP32-S3-WROOM-1-N8R8 |
| **封装** | WROOM-1 模块 |
| **Flash** | 8 MB |
| **PSRAM** | 8 MB（可选） |

### 3.2 决策理由

选择 ESP32-S3 的核心理由：

| 理由 | 说明 |
|------|------|
| **USB 内置** | 内置 USB OTG，无需外接 USB 芯片，减少硬件复杂度和成本 |
| **BLE 内置** | 内置 BLE 5.0，支持 FIDO GATT 服务，无需外接 BLE 模块 |
| **成本适中** | 单价 $5-8，符合项目成本目标 |
| **开发便捷** | ESP-IDF 官方 SDK 完善，PlatformIO 支持，开发门槛低 |
| **社区活跃** | ESP32 社区资源丰富，文档完善，问题易解决 |
| **开源参考** | SoloKeys、OpenSK 等项目提供成熟架构参考 |
| **性能充足** | 双核 240MHz，512KB RAM，满足 FIDO2 协议栈需求 |
| **存储足够** | 8MB 内置 Flash，满足凭证存储需求 |

## 四、考虑的选项

### 4.1 候选芯片对比

| 芯片 | 架构 | 主频 | Flash | RAM | USB | BLE | 价格 | 开发难度 |
|------|------|------|-------|-----|-----|-----|------|----------|
| **ESP32-S3** | Xtensa LX7 | 240 MHz | 8 MB | 512 KB | 内置 | 内置 | $5-8 | 低 |
| **nRF52840** | Cortex-M4F | 64 MHz | 1 MB | 256 KB | 内置 | 内置 | $5-7 | 中 |
| **RP2040** | Cortex-M0+ | 133 MHz | 外部 | 264 KB | 内置 | 无 | $1-2 | 低 |
| **STM32L4** | Cortex-M4 | 80 MHz | 1 MB | 128 KB | 内置 | 无 | $5-8 | 中 |

### 4.2 详细对比分析

#### ESP32-S3

| 项目 | 说明 |
|------|------|
| **优点** | USB + BLE 内置、成本适中、社区活跃、开发工具完善、双核高性能 |
| **缺点** | 功耗较高（~100mA）、无硬件安全模块、需要外接安全芯片 |
| **适用性** | **推荐！USB + BLE 双模认证器最佳选择** |
| **备注** | ESP-IDF 和 PlatformIO 支持完善 |

#### nRF52840

| 项目 | 说明 |
|------|------|
| **优点** | BLE + USB 内置、低功耗、硬件安全特性、专业级方案 |
| **缺点** | 价格较高、开发工具复杂、社区资源较少、学习曲线高 |
| **适用性** | 适合专业级 BLE + USB 双模认证器 |
| **备注** | Nordic SDK 完善，OpenSK 项目使用 |

#### RP2040

| 项目 | 说明 |
|------|------|
| **优点** | 价格极低（$1-2）、USB 内置、社区活跃、开发门槛最低 |
| **缺点** | 无 BLE 内置（需外接）、无硬件安全、需要外部 Flash |
| **适用性** | 适合极低成本 USB 认证器 |
| **备注** | 不适合 BLE 双模场景 |

#### STM32L4

| 项目 | 说明 |
|------|------|
| **优点** | 超低功耗、硬件安全特性（部分型号）、丰富的外设 |
| **缺点** | 价格较高、无 BLE 内置、开发工具复杂（ST-Link） |
| **适用性** | 适合低功耗 USB/NFC 认证器 |
| **备注** | 不适合 BLE 双模场景 |

### 4.3 功能对比矩阵

| 功能需求 | ESP32-S3 | nRF52840 | RP2040 | STM32L4 |
|----------|----------|----------|---------|---------|
| **USB HID** | 内置 | 内置 | 内置 | 内置 |
| **BLE 5.0** | 内置 | 内置 | 无 | 无 |
| **双核处理** | 支持 | 单核 | 双核 | 单核 |
| **硬件加密** | 无 | 有 | 无 | 部分 |
| **UART 接口** | 多个 | 多个 | 多个 | 多个 |
| **I2C 接口** | 多个 | 多个 | 多个 | 多个 |
| **SPI 接口** | 多个 | 多个 | 多个 | 多个 |
| **Flash 存储** | 8MB | 1MB | 外部 | 1MB |
| **Secure Boot** | 支持 | 支持 | 无 | 支持 |
| **社区支持** | 极高 | 中 | 高 | 中 |
| **开发便捷** | 高 | 中 | 高 | 中 |

### 4.4 对比结论

| 评估维度 | ESP32-S3 | nRF52840 | RP2040 | STM32L4 |
|----------|----------|----------|---------|---------|
| **USB+BLE 双模** | 最佳 | 最佳 | 不适合 | 不适合 |
| **成本效益** | 最佳 | 中 | 极佳 | 中 |
| **开发便捷** | 最佳 | 中 | 最佳 | 中 |
| **功耗** | 中 | 最佳 | 中 | 最佳 |
| **安全性** | 中（需外接） | 最佳 | 中（需外接） | 中 |
| **综合评分** | **第一选择** | 第二选择 | 第三选择 | 第四选择 |

**选择 ESP32-S3 的核心优势**：
1. USB + BLE 双模内置，无需外接芯片
2. 成本适中，符合 DIY 项目预算
3. 开发便捷，ESP-IDF + PlatformIO 完善
4. 社区活跃，问题解决效率高

## 九、替代方案分析

### 9.1 替代方案一：nRF52840

**方案描述**：
- 主控：nRF52840
- 特点：低功耗、硬件安全、专业级
- 参考：OpenSK 项目

**对比分析**：

| 项目 | nRF52840 | ESP32-S3 |
|------|----------|----------|
| **USB + BLE** | 内置 | 内置 |
| **功耗** | 低（优） | 中 |
| **硬件安全** | 有（优） | 无 |
| **开发复杂度** | 中 | 低（优） |
| **成本** | $5-7 | $5-8 |
| **社区支持** | 中 | 高（优） |

**未选择原因**：
1. 开发工具复杂，学习曲线高
2. Nordic SDK 不如 ESP-IDF 完善
3. 社区资源较少，问题解决效率低
4. 对于 DIY 项目，ESP32-S3 开发门槛更低

### 9.2 替代方案二：RP2040

**方案描述**：
- 主控：RP2040
- 特点：极低成本、USB 内置、社区活跃
- 成本：$1-2

**对比分析**：

| 项目 | RP2040 | ESP32-S3 |
|------|----------|----------|
| **成本** | 极低（优） | 中 |
| **USB 内置** | 有 | 有 |
| **BLE 内置** | 无 | 有（优） |
| **硬件安全** | 无 | 无 |
| **社区支持** | 高 | 高 |
| **开发便捷** | 高 | 高 |

**未选择原因**：
1. 无 BLE 内置，需外接 BLE 模块，增加成本和复杂度
2. 需要外部 Flash，增加硬件复杂度
3. 不符合 USB + BLE 双模项目需求

### 9.3 替代方案三：STM32L4

**方案描述**：
- 主控：STM32L4
- 特点：超低功耗、硬件安全（部分型号）
- 适用：低功耗场景

**对比分析**：

| 项目 | STM32L4 | ESP32-S3 |
|------|----------|----------|
| **功耗** | 极低（优） | 中 |
| **USB 内置** | 有（部分） | 有 |

| **硬件安全** | 有（部分） | 无 |
| **开发复杂度** | 中 | 低（优） |
| **成本** | $5-8 | $5-8 |

**未选择原因**：
1. 无 BLE 内置，需外接 BLE 模块
2. 开发工具复杂（ST-Link、CubeMX）
3. 不符合 USB + BLE 双模项目需求
