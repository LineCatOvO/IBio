# ADR-007: 选择 ESP32-S3 内置 Flash + ATECC608A 密钥存储方案

**状态**：已接受（Accepted）

**决策日期**：2026-04-06

**决策者**：IBio 项目团队

---

## 一、标题

选择 ESP32-S3 内置 8MB Flash + ATECC608A 密钥存储方案作为 IBio FIDO2 安全密钥的存储架构

---

## 二、背景

### 2.1 项目存储需求

IBio 项目作为一个 FIDO2 安全密钥，需要存储以下类型的数据：

| 数据类型 | 存储内容 | 存储要求 |
|----------|----------|----------|
| **固件代码** | 系统固件、应用程序、库代码 | 非易失性、可更新、可加密 |
| **凭证数据** | Resident credentials、凭证 ID 映射表 | 非易失性、可加密、容量适中 |
| **配置数据** | 设备配置、用户偏好、PIN 配置 | 非易失性、易读写、容量小 |
| **密钥数据** | 私钥、attestation 密钥、PIN 密钥 | 安全存储、不可导出、硬件保护 |
| **指纹模板** | 指纹特征模板（FPM383C 内置） | 内置存储、无需外部存储 |

### 2.2 FIDO2 存储要求

根据 FIDO2/CTAP2 标准，认证器需要满足以下存储要求：

| FIDO2 要求 | 具体内容 | 存储影响 |
|------------|----------|----------|
| **Resident Credential** | 凭证可存储在认证器本地 | 需存储凭证 ID + 元数据 |
| **凭证数量** | 最少 8 个 resident credential（推荐 100+） | 需足够容量存储多个凭证 |
| **凭证大小** | 每个凭证约 100-200 字节（不含私钥） | 总容量需 10-20KB+ |
| **私钥存储** | 私钥必须不可导出、安全存储 | 必须使用安全芯片或加密存储 |
| **PIN 存储** | PIN 相关数据需加密存储 | 需安全存储 PIN hash 和计数器 |
| **Attestation 密钥** | 需存储 attestation 私钥和证书 | 需安全芯片存储 |

### 2.3 技术约束

| 约束条件 | 说明 |
|----------|------|
| **主控芯片** | ESP32-S3-WROOM-1-N8R8（内置 8MB Flash） |
| **安全芯片** | ATECC608A（16 个密钥槽位） |
| **指纹模块** | FPM383C（内置 200 个指纹模板存储） |
| **成本约束** | 存储成本最小化（目标是零额外存储成本） |
| **OTA 需求** | 需支持固件 OTA 更新，需要预留 OTA 空间 |

### 2.4 决策触发

需要确定存储方案以满足：
- 固件存储和 OTA 更新需求
- FIDO2 凭证存储需求（resident credential）
- 密钥安全存储需求（私钥不可导出）
- 设备配置存储需求
- 成本最小化目标

---

## 三、决策

### 3.1 最终决策

选择以下存储方案：

| 存储类型 | 存储介质 | 具体方案 |
|----------|----------|----------|
| **固件存储** | ESP32-S3 内置 Flash | 8MB 内置 Flash（固件分区） |
| **凭证存储** | ESP32-S3 内置 Flash | Flash 凭证区（加密存储） |
| **配置存储** | ESP32-S3 内置 Flash | NVS 分区（非易失性存储） |
| **密钥存储** | ATECC608A 安全芯片 | 16 个密钥槽位（硬件保护） |
| **指纹模板** | FPM383C 内置存储 | 传感器内部存储（200 个模板） |

### 3.2 决策理由

选择该存储方案的核心理由：

1. **成本最优**：ESP32-S3 内置 8MB Flash，无需外接存储芯片，存储成本为零
2. **容量充足**：8MB Flash 可满足固件 + OTA + 凭证 + 配置所有需求
3. **安全合规**：ATECC608A 提供硬件级密钥保护，私钥不可导出，符合 FIDO2 要求
4. **架构清晰**：固件、凭证、配置分区分离，便于管理和维护
5. **OTA 支持**：ESP-IDF 提供完善的 OTA 分区和更新机制

## 四、考虑的选项

### 4.1 Flash 存储选项对比

| 选项 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| **ESP32-S3 内置 Flash** | 成本为零、容量足够、内置集成 | 不可扩展、寿命有限 | $0 |
| **外接 W25Q64（8MB）** | 可扩展、可更换 | 需额外 PCB 空间、增加成本 | $1-2 |
| **外接 W25Q128（16MB）** | 更大容量、可扩展 | 成本更高、空间占用 | $1-2 |
| **内置 + 外接组合** | 容量最大、灵活性高 | 成本最高、复杂度高 | $1-2 |

**对比结论**：
- ESP32-S3 内置 8MB Flash 容量足够，成本最优
- 外接 Flash 仅在需要更大容量时考虑
- 内置 + 外接组合方案复杂度高，不推荐

### 4.2 凭证存储选项对比

| 选项 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| **内置 Flash 加密存储** | 成本为零、容量足够 | 需软件加密、安全性较低 | $0 |
| **EEPROM（AT24C64）** | 易读写、寿命长 | 容量小、成本增加 | $0.5-1 |
| **FRAM（FM25W256）** | 高速度、高寿命、低功耗 | 成本高、容量有限 | $3-5 |
| **ATECC608A 密钥槽位** | 硬件安全、不可导出 | 槽位有限（仅 16 个） | $1-2 |

**对比结论**：
- 内置 Flash 加密存储成本最优，容量足够
- EEPROM 容量小，不适合凭证存储
- FRAM 成本高，不适合 DIY 项目
- ATECC608A 仅用于密钥存储，不适合凭证元数据

### 4.3 密钥存储选项对比

| 选项 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| **ATECC608A** | 硬件安全、不可导出、开源验证 | 槽位有限（16 个） | $1-2 |
| **ESP32-S3 Flash 加密** | 成本为零 | 无硬件保护、安全性较低 | $0 |
| **软件加密存储** | 成本为零、灵活性高 | 无硬件保护、安全性最低 | $0 |

**对比结论**：
- ATECC608A 硬件安全最优，符合 FIDO2 要求
- ESP32-S3 Flash 加密安全性较低，不推荐用于私钥
- 软件加密存储安全性最低，禁止用于私钥

### 4.4 存储方案综合对比

| 方案 | 固件存储 | 凭证存储 | 密钥存储 | 总成本 | 推荐度 |
|------|----------|----------|----------|--------|--------|
| **方案 A：内置 + ATECC608A** | 内置 Flash | 内置 Flash（加密） | ATECC608A | $1-2 | 推荐 |
| **方案 B：内置 + 软件加密** | 内置 Flash | 内置 Flash（加密） | 内置 Flash（加密） | $0 | 不推荐 |
| **方案 C：外接 + ATECC608A** | 外接 Flash | 外接 Flash（加密） | ATECC608A | $2-4 | 备选 |
| **方案 D：内置 + EEPROM** | 内置 Flash | EEPROM | ATECC608A | $1.5-3 | 备选 |

**最终选择**：方案 A（内置 Flash + ATECC608A）
- 成本最优（仅需 ATECC608A 成本）
- 安全合规（ATECC608A 硬件保护）
- 容量充足（8MB 内置 Flash）

## 五、技术规格

### 5.1 ESP32-S3 内置 Flash 规格

| 项目 | 规格 |
|------|------|
| **Flash 类型** | SPI NOR Flash |
| **容量** | 8 MB |
| **接口** | SPI（内置，四线模式） |
| **速度** | 读取 ~80MB/s，写入 ~40MB/s（四线模式） |
| **分区** | 可自定义分区表 |
| **加密** | 支持 Flash Encryption（可选） |
| **寿命** | 10,000+ 擦写周期 |

### 5.2 Flash 分区方案

ESP32-S3 8MB Flash 分区表设计：

| 分区名称 | 分区类型 | 子类型 | 偏移地址 | 大小 | 用途 |
|----------|----------|--------|----------|------|------|
| **nvs** | data | nvs | 0x9000 | 24 KB (0x6000) | 配置存储 |
| **otadata** | data | ota | 0xf000 | 8 KB (0x2000) | OTA 状态 |
| **phy_init** | data | phy | 0x11000 | 4 KB (0x1000) | RF 初始化 |
| **factory** | app | factory | 0x20000 | 1 MB (0x100000) | 原厂固件 |
| **ota_0** | app | ota_0 | 0x120000 | 1 MB (0x100000) | OTA 固件 A |
| **ota_1** | app | ota_1 | 0x220000 | 1 MB (0x100000) | OTA 固件 B |
| **credential** | data | spiffs | 0x320000 | 512 KB (0x80000) | 凭证存储 |
| **storage** | data | spiffs | 0x3A0000 | 4.5 MB (0x480000) | 扩展存储 |
| **总计** | - | - | 0x0 | 8 MB | 全部 Flash |

### 5.3 ATECC608A 密钥存储规格

| 项目 | 规格 |
|------|------|
| **芯片型号** | ATECC608A-MAHDA-T |
| **接口** | I2C（最高 1 MHz） |
| **密钥槽位** | 16 个 |
| **密钥类型** | ES256（ECDSA P-256）、ECDH、AES-128 |
| **安全等级** | EAL4+ |
| **密钥保护** | 私钥不可导出、硬件加密存储 |
| **防篡改** | 防侧信道攻击、温度检测 |

### 5.4 密钥槽位分配

ATECC608A 密钥槽位分配方案：

| 槽位 | 用途 | 密钥类型 | 权限配置 |
|------|------|----------|----------|
| **Slot 0** | 凭证密钥 1 | ES256 私钥 | 不可读、可签名 |
| **Slot 1** | 凭证密钥 2 | ES256 私钥 | 不可读、可签名 |
| **Slot 2** | 凭证密钥 3 | ES256 私钥 | 不可读、可签名 |
| **Slot 3** | 凭证密钥 4 | ES256 私钥 | 不可读、可签名 |
| **Slot 4** | 凭证密钥 5 | ES256 私钥 | 不可读、可签名 |
| **Slot 5** | 凭证密钥 6 | ES256 私钥 | 不可读、可签名 |
| **Slot 6** | 凭证密钥 7 | ES256 私钥 | 不可读、可签名 |
| **Slot 7** | 凭证密钥 8 | ES256 私钥 | 不可读、可签名 |
| **Slot 8-9** | 凭证密钥扩展 | ES256 私钥 | 保留扩展 |
| **Slot 10-13** | 保留槽位 | - | 保留备用 |
| **Slot 14** | attestation 密钥 | ES256 私钥 | 不可读、可签名 |
| **Slot 15** | PIN 密钥 | ECDH 密钥 | 不可读、可 ECDH |

### 5.5 FPM383C 指纹模板存储规格

### 7.1 存储需求分析

| 数据类型 | 存储需求 | 容量估算 |
|----------|----------|----------|
| **固件代码** | 系统固件 + 应用代码 | ~1.5 MB |
| **OTA 固件** | OTA 更新备份固件 | ~1.5 MB（双备份） |
| **凭证数据** | Resident credentials（100 个） | ~100 KB |
| **凭证映射表** | Credential ID 映射表 | ~10 KB |
| **配置数据** | 设备配置、NVS 数据 | ~24 KB |
| **PIN 数据** | PIN hash、计数器 | ~1 KB |
| **指纹模板** | 指纹特征模板 | FPM383C 内部存储 |
| **密钥数据** | 私钥（最多 16 个） | ATECC608A 内部存储 |

### 7.2 固件容量分析

| 固件组件 | 估算大小 |
|----------|----------|
| **ESP-IDF 基础库** | ~500 KB |
| **FreeRTOS 内核** | ~50 KB |
| **USB HID 驱动** | ~30 KB |
| **BLE GATT 服务** | ~100 KB |
| **CTAP 协议栈** | ~200 KB |
| **ATECC608A 驱动** | ~50 KB |
| **FPM383C 驱动** | ~30 KB |
| **加密库** | ~50 KB |
| **应用逻辑** | ~100 KB |
| **总计** | **~1.1 MB** |

预留空间：1 MB - 1.1 MB = 0.9 MB（足够余量）

### 7.3 凭证容量分析

**单个 Resident Credential 存储结构**：

| 字段 | 大小 | 说明 |
|------|------|------|
| **Credential ID** | 32 字节 | 凭证唯一标识 |
| **RP ID Hash** | 32 字节 | 依赖方 ID 的 SHA-256 哈希 |
| **用户信息** | 64 字节 | 用户名、显示名等 |
| **密钥槽位索引** | 1 字节 | ATECC608A 槽位索引 |
| **标志位** | 4 字节 | UV、RK 等标志 |
| **计数器** | 4 字节 | 签名计数器 |
| **扩展数据** | 32 字节 | 可选扩展数据 |
| **总计** | **~137 字节/凭证** |

**凭证容量估算**：
- 目标凭证数量：100 个 resident credential
- 单个凭证大小：~137 字节
- 总凭证容量：100 × 137 = 13.7 KB
- 预留空间：凭证分区 512 KB，足够存储 100+ 凭证

### 7.4 密钥容量分析

**ATECC608A 密钥槽位分析**：

| 密钥类型 | 槽位需求 | 可用槽位 | 容量评估 |
|----------|----------|----------|----------|
| **凭证密钥** | 每个 credential 1 个槽位 | Slot 0-9（10 个） | 支持 10 个凭证密钥 |
| **Attestation 密钥** | 1 个槽位 | Slot 14 | 支持设备 attestation |
| **PIN 密钥** | 1 个槽位 | Slot 15 | 支持 PIN 加密通道 |
| **总计** | - | 12 个可用槽位 | 支持 10 个 resident credential |

**密钥容量限制**：
- ATECC608A 仅支持 16 个密钥槽位
- 分配 10 个槽位给凭证密钥
- 支持 10 个 resident credential（最小满足 FIDO2 要求）
- 扩展方案：超过 10 个凭证时使用加密存储 + ATECC608A 签名

### 7.5 总容量评估

| 存储区域 | 分配容量 | 需求容量 | 余量 |
|----------|----------|----------|------|
| **固件区（factory + ota_0 + ota_1）** | 3 MB | 3 MB（固件 + OTA） | 0 MB |
| **凭证区** | 512 KB | 100 KB（100 凭证） | 412 KB |
| **配置区（nvs）** | 24 KB | 24 KB（配置） | 0 KB |
| **扩展存储区** | 4.5 MB | 0 KB（预留） | 4.5 MB |
| **总计** | 8 MB | 3.124 MB | **4.876 MB 余量** |

**结论**：ESP32-S3 内置 8MB Flash 容量充足，满足所有存储需求。

|----------|----------|----------|----------|
| **固件代码** | Flash factory/ota 分区 | 明文（可选加密） | Flash Encryption（可选） |
| **OTA 状态** | Flash otadata 分区 | 明文 | 无加密需求 |
| **设备配置** | Flash NVS 分区 | NVS 加密（可选） | NVS 加密（可选） |
| **凭证元数据** | Flash credential 分区 | 加密存储 | AES-128 加密 |
| **凭证私钥** | ATECC608A Slot 0-9 | 硬件存储 | 不可导出、硬件保护 |
| **Attestation 密钥** | ATECC608A Slot 14 | 硋件存储 | 不可导出、硬件保护 |
| **PIN 密钥** | ATECC608A Slot 15 | 硬件存储 | 不可导出、ECDH 保护 |
| **指纹模板** | FPM383C 内部 | 内部存储 | 模板不可导出 |

### 8.3 凭证存储加密方案

**凭证存储加密架构**：

```
┌─────────────────────────────────────────────────────────────────┐
│                   凭证存储加密流程                               │
└─────────────────────────────────────────────────────────────────┘

    明文凭证数据
         │
         ▼
    ┌─────────────┐
    │ AES-128 加密│（密钥存储在 NVS 或 ATECC608A）
    └─────────────┘
         │
         ▼
    加密凭证数据
         │
         ▼
    ┌─────────────┐
    │ Flash 写入  │（credential 分区）
    └─────────────┘
         │
         ▼
    Flash 存储（加密态）
```

**凭证加密方案选择**：
- **方案 A**：使用 ATECC608A 生成随机密钥，存储在 NVS
- **方案 B**：使用 ESP32-S3 Flash Encryption 密钥（推荐）
- **方案 C**：使用固定密钥（不推荐）

**推荐方案 B**：
- 利用 ESP32-S3 Flash Encryption 内置密钥
- 无需额外密钥管理
- 硬件级加密保护

## 九、风险评估

### 9.1 Flash 容量不足风险

| 风险描述 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **固件体积增大超出预期** | 低 | 高 | 优化编译选项、裁剪功能、使用外接 Flash |
| **凭证数量超过预期** | 低 | 中 | 凭证分区预留 512 KB，足够存储 100+ 凭证 |
| **OTA 空间不足** | 低 | 高 | OTA 分区预留 2 MB，足够固件更新 |

**缓解措施**：
- 固件体积监控：定期检查编译后固件大小
- OTA 空间预留：factory + ota_0 + ota_1 共 3 MB
- 凭证数量限制：默认支持 10 个 resident credential（ATECC608A 限制）

### 9.2 凭证存储安全风险

| 风险描述 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **Flash 数据泄露** | 中 | 高 | 使用 Flash Encryption、凭证数据加密存储 |
| **私钥导出** | 低 | 极高 | ATECC608A 硬件保护，私钥不可导出 |
| **凭证篡改** | 中 | 高 | 凭证数据加密存储、签名验证 |
| **PIN 泄露** | 低 | 高 | PIN 使用 ECDH 加密通道、ATECC608A 保护 |

**缓解措施**：
- 启用 ESP32-S3 Flash Encryption
- 凭证数据 AES-128 加密存储
- ATECC608A 私钥硬件保护
- PIN 验证使用 ECDH 加密通道

### 9.3 Flash 寿命风险

| 风险描述 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **Flash 擦写次数超限** | 低 | 高 | 减少写入频率、磨损均衡、使用 NVS |
| **凭证频繁更新导致寿命耗尽** | 低 | 中 | 凭证分区使用磨损均衡算法 |

**缓解措施**：
- 使用 NVS 磨损均衡机制
- 凭证分区减少写入频率
- 避免频繁 OTA 更新
- Flash 寿命 10,000+ 擦写周期，足够使用

### 9.4 OTA 失败风险

| 风险描述 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **OTA 更新失败导致固件损坏** | 中 | 极高 | 双 OTA 分区备份、回滚机制 |
| **OTA 下载中断** | 低 | 高 | 固件签名验证、完整性校验 |
| **OTA 分区损坏** | 低 | 高 | factory 分区保留原厂固件 |

**缓解措施**：
- 使用双 OTA 分区（ota_0 + ota_1）
- OTA 回滚机制：失败后回滚到旧固件
- factory 分区保留原厂固件
- 固件签名验证：防止篡改固件

### 9.5 ATECC608A 密钥槽位不足风险

| 风险描述 | 可能性 | 影响 | 缓解措施 |
|----------|--------|------|----------|
| **密钥槽位不足（超过 10 个凭证）** | 中 | 中 | 使用加密存储 + ATECC608A 签名方案 |
| **密钥槽位损坏** | 低 | 高 | 密钥槽位配置锁定、防篡改 |

**缓解措施**：
- 默认支持 10 个 resident credential
- 扩展方案：超过 10 个凭证时使用加密存储私钥 + ATECC608A 签名
- ATECC608A 密钥槽位配置锁定

## 十一、替代方案分析

### 11.1 替代方案一：外接 Flash + ATECC608A

**方案描述**：
- 固件存储：外接 W25Q128（16MB Flash）
- 凭证存储：外接 Flash
- 密钥存储：ATECC608A

**对比分析**：

| 项目 | 外接 Flash + ATECC608A | 内置 Flash + ATECC608A |
|------|------------------------|------------------------|
| **容量** | 16 MB（更大） | 8 MB（足够） |
| **成本** | $2-4（增加 $1-2） | $1-2（最优） |
| **复杂度** | 高（需外接芯片） | 低（内置集成） |
| **PCB 空间** | 需额外空间 | 无额外空间 |
| **OTA 支持** | 支持 | 支持 |

**未选择原因**：
1. 内置 8MB Flash 容量已足够
2. 外接 Flash 增加成本和复杂度
3. 外接 Flash 需额外 PCB 空间
4. 内置 Flash 性能更优（内置集成）

### 11.2 替代方案二：内置 Flash + EEPROM

**方案描述**：
- 固件存储：ESP32-S3 内置 Flash
- 凭证存储：EEPROM AT24C128（128KB）
- 密钥存储：ATECC608A

**对比分析**：

| 项目 | 内置 Flash + EEPROM | 内置 Flash（凭证区） |
|------|---------------------|---------------------|
| **凭证存储** | EEPROM（独立） | Flash 分区 |
| **容量** | 128 KB EEPROM | 512 KB Flash 分区 |
| **成本** | $1.5-3（增加 $0.5-1） | $1-2（最优） |
| **复杂度** | 中（需 EEPROM 驱动） | 低（使用 SPIFFS） |
| **寿命** | EEPROM 寿命更长 | Flash 寿命有限 |

**未选择原因**：
1. EEPROM 容量有限（128KB），不如 Flash 分区（512KB）
2. EEPROM 需额外驱动和 I2C 接口
3. EEPROM 增加成本（$0.5-1）
4. 内置 Flash 分区足够满足凭证存储需求

### 11.3 替代方案三：FRAM 高可靠性方案

**方案描述**：
- 固件存储：ESP32-S3 内置 Flash
- 配置存储：FRAM FM25W256（256KB）
- 密钥存储：ATECC608A

**对比分析**：

| 项目 | FRAM 方案 | 内置 Flash 方案 |
|------|-----------|-----------------|
| **可靠性** | 极高（FRAM 寿命无限） | 高（Flash 寿命有限） |
| **速度** | 极快（FRAM 快速写入） | 中（Flash 写入慢） |
| **成本** | $4-7（增加 $3-5） | $1-2（最优） |
| **容量** | 256 KB FRAM | 512 KB Flash 分区 |

**未选择原因**：
1. FRAM 成本高（$3-5），不适合 DIY 项目
2. FRAM 容量有限（256KB），不如 Flash 分区
3. Flash 寿命已足够（10,000+ 擦写）
4. DIY 项目不要求极高可靠性

### 11.4 替代方案四：纯软件加密方案（无 ATECC608A）

**方案描述**：
- 固件存储：ESP32-S3 内置 Flash
- 凭证存储：内置 Flash 加密存储
- 密钥存储：Flash 加密存储（软件加密）

**对比分析**：

| 项目 | 软件加密方案 | ATECC608A 方案 |
|------|--------------|-----------------|
| **成本** | $0（最低） | $1-2（增加 ATECC608A） |
| **安全性** | 低（无硬件保护） | 高（硬件保护） |
| **私钥保护** | 软件加密（可导出） | 不可导出 |
| **FIDO2 合规** | 不符合（私钥可导出） | 符合（私钥不可导出） |

**未选择原因**：
1. 不符合 FIDO2 私钥不可导出要求
2. 安全性低，无硬件保护
3. 软件加密密钥可被导出或破解
4. FIDO2 认证测试可能失败
