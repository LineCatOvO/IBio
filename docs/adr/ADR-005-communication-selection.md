# ADR-005: 选择 USB + BLE 双模通信方案

**状态**：已接受（Accepted）

**决策日期**：2026-04-06

**决策者**：IBio 项目团队

---

## 一、标题

选择 ESP32-S3 内置 USB OTG + BLE 5.0 双模通信方案实现 IBio FIDO2 安全密钥

---

## 二、背景

### 2.1 项目需求

IBio 项目作为一个 FIDO2 安全密钥，需要满足以下通信需求：

| 需求类型 | 具体需求 | 来源 |
|----------|----------|------|
| **USB 支持** | USB HID 设备模式，支持 CTAP over HID | FIDO2 CTAP 规范 |
| **BLE 支持** | BLE GATT 服务，支持 CTAP over BLE | FIDO2 CTAP 规范 |
| **双模支持** | USB 和 BLE 同时支持，无缝切换 | WebAuthn 用户场景 |
| **成本最小化** | 通信模块成本应最小化，避免外接芯片 | 项目成本目标 $20-31 |

### 2.2 技术约束

| 约束条件 | 说明 |
|----------|------|
| **USB HID 协议** | 必须符合 CTAP over HID 规范，使用 Interrupt 端点 |
| **BLE GATT 服务** | 必须符合 FIDO GATT 服务定义（UUID: 0xFFFD） |
| **传输协议** | 必须实现 CTAP 传输绑定协议（分片、重组、超时） |
| **硬件兼容** | 通信方案需与 ESP32-S3 主控芯片兼容 |
| **开发时间** | DIY 项目，开发周期不宜过长 |

### 2.3 成本约束

| 成本项 | 目标成本 | 说明 |
|----------|----------|------|
| **USB 芯片** | $0（内置） | 避免外接 USB 芯片 |
| **BLE 芯片** | $0（内置） | 避免外接 BLE 模块 |
| **连接器** | $0.5-1 | USB-C 连接器成本 |
| **总计** | $0.5-1 | 仅连接器成本 |

### 2.4 FIDO2 传输协议要求

**CTAP over HID 要求**：

| 项目 | 规格 | 说明 |
|------|------|------|
| **接口类型** | USB HID | Human Interface Device |
| **端点类型** | Interrupt IN/OUT | 中断传输 |
| **数据包大小** | 64 字节 | HID 报告大小 |
| **最大响应时间** | 30 秒 | 响应超时时间 |
| **序列号** | 必须 | 设备序列号 |
| **Wink 功能** | 推荐 | 触觉/视觉反馈 |

**CTAP over BLE 要求**：

| 项目 | 规格 | 说明 |
|------|------|------|
| **BLE 版本** | BLE 4.2+ | 推荐 BLE 5.0 |
| **GATT 服务** | FIDO Service | UUID: 0xFFFD |
| **特征值** | Control Point, Status, Service Revision | 必须特征值 |
| **连接间隔** | 15-30 ms | 推荐间隔 |
| **MTU** | 最小 20 字节 | 推荐 64+ |
| **超时时间** | 30 秒 | 响应超时 |

### 2.5 决策触发

需要确定通信方案以满足 FIDO2 双模需求，并确保：
- USB HID 设备模式支持
- BLE GATT 服务支持
- 成本最小化（无外接芯片）
- 开发可行性（ESP-IDF 支持）

## 三、决策

### 3.1 最终决策

选择以下通信方案：

| 类别 | 选择 | 具体方案 |
|------|------|----------|
| **USB** | ESP32-S3 内置 USB OTG | USB HID 设备模式 |
| **BLE** | ESP32-S3 内置 BLE 5.0 | FIDO GATT 服务 |
| **USB 连接器** | USB-C | Type-C 连接器 |
| **传输协议** | CTAP over HID/BLE | ESP-IDF + 自定义实现 |

### 3.2 决策理由

选择该方案的核心理由：

1. **内置 USB + BLE**：ESP32-S3 内置 USB OTG 和 BLE 5.0，无需外接 USB/BLE 芯片，显著降低成本和复杂度
2. **成本最低**：通信模块成本仅 USB-C 连接器（$0.5-1），无额外芯片成本
3. **ESP-IDF 驱动完善**：ESP-IDF 提供完善的 USB HID 和 BLE GATT 示例，开发可行性高
4. **SoloKeys 参考架构**：SoloKeys 项目使用类似架构，有成熟参考
5. **双模无缝支持**：USB 和 BLE 同时支持，满足 FIDO2 双模需求

## 四、考虑的选项

### 4.1 内置 vs 外接芯片对比

| 选项 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| **ESP32-S3 内置 USB + BLE** | 无外接芯片、成本最低、ESP-IDF 支持 | 功耗较高、无硬件安全 | $0 |
| **外接 USB 芯片 + 内置 BLE** | USB 独立、稳定性好 | 成本增加、PCB 复杂 | $0.5-2 |
| **内置 USB + 外接 BLE 模块** | BLE 独立、选择灵活 | 成本增加、功耗增加 | $3-10 |
| **外接 USB + 外接 BLE** | 灵活选择、稳定可靠 | 成本最高、PCB 最复杂 | $3-12 |

**对比结论**：
- ESP32-S3 内置方案成本最优，无外接芯片
- 外接芯片方案成本显著增加，不推荐

**选择内置方案理由**：
1. 成本最低，符合项目目标
2. 无外接芯片，简化 PCB 设计
3. ESP-IDF 驱动完善，开发可行
4. 双模无缝支持

### 4.2 通信模式对比

| 模式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **USB only** | 成本低、稳定、无功耗问题 | 无无线支持、便携性差 | 固定场景认证器 |
| **BLE only** | 无线便携、低功耗 | 无 USB 支持、依赖蓝牙 | 移动场景认证器 |
| **USB + BLE** | 双模支持、场景广泛 | 成本增加、功耗增加 | 通用认证器 |
| **USB + NFC + BLE** | 全协议支持、最大兼容性 | 成本最高、复杂度高 | 企业级认证器 |

**对比结论**：
- USB + BLE 双模满足 FIDO2 主流需求
- USB only 缺少便携性
- BLE only 缺少桌面支持
- USB + NFC + BLE 成本过高（$34-51）

**选择 USB + BLE 理由**：
1. 双模支持满足主流场景
2. 成本适中（$20-31）
3. 开发复杂度可控
4. 参考 SoloKeys 架构

### 4.3 USB 方案对比

| 方案 | 芯片 | 接口 | 成本 | 适用性 |
|------|------|------|------|--------|
| **ESP32-S3 内置 USB OTG** | 内置 | USB HID | $0 | 推荐！无需外接 |
| **CH340G** | UART 转 USB | UART | $0.5-1 | 低成本外接 |
| **CP2102** | UART 转 USB | UART | $1-2 | 稳定性较好 |
| **FT232RL** | UART 转 USB | UART | $2-3 | 工业级 |
| **nRF52840 内置 USB** | 内置 | USB HID | $0 | Nordic 方案 |
| **RP2040 内置 USB** | 内置 | USB HID | $0 | 低成本方案 |

**对比结论**：
- 内置 USB 方案成本最优
- ESP32-S3 USB OTG 支持 HID 设备模式
- UART 转 USB 芯片不适合 HID 设备

**选择 ESP32-S3 内置 USB 理由**：
1. 成本 $0，无外接芯片
2. USB HID 设备模式支持
3. ESP-IDF USB HID 示例完善
4. SoloKeys 项目验证可用

### 4.4 BLE 方案对比

| 方案 | 芯片 | BLE 版本 | 成本 | 适用性 |
|------|------|----------|------|--------|
| **ESP32-S3 内置 BLE** | 内置 | BLE 5.0 | $0 | 推荐！无需外接 |
| **nRF52840 内置 BLE** | 内置 | BLE 5.0 | $0 | Nordic 方案 |
| **CC2541** | 外接 | BLE 4.0 | $3-5 | 旧方案 |
| **Nina B302** | 外接模块 | BLE 5.0 | $8-12 | 模块化方案 |
| **Nina B112** | 外接模块 | BLE 5.0 | $6-10 | 低成本模块 |

**对比结论**：
- 内置 BLE 方案成本最优
- ESP32-S3 BLE 5.0 支持 FIDO GATT 服务
- 外接 BLE 模块成本显著增加

**选择 ESP32-S3 内置 BLE 理由**：
1. 成本 $0，无外接芯片
2. BLE 5.0 支持，满足 FIDO 要求
3. ESP-IDF BLE GATT 示例完善
4. OpenSK 项目验证可用

## 八、协议实现方案

### 8.1 USB HID 协议实现方案

**ESP-IDF USB HID 实现路径**：

| 步骤 | 实现内容 | ESP-IDF API |

| **4. HID 数据收发** | 实现 HID 数据收发 | `tud_hid_get_report_cb()` |
| **5. CTAP 消息处理** | 实现 CTAP over HID 协议 | 自定义实现 |

- `esp-idf/examples/peripherals/usb/device/tusb_hid`

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **主机不识别 HID 设备** | 低 | 高 | 使用 ESP-IDF 标准 HID 示例 |
| **Windows/Mac/Linux 兼容性问题** | 中 | 高 | 多平台测试验证 |
| **USB 连接稳定性问题** | 低 | 中 | 优化 USB 端点配置 |
| **HID 描述符错误** | 低 | 高 | 参考 FIDO2 HID 描述符标准 |

### 9.2 BLE GATT 稳定性风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **BLE 连接不稳定** | 中 | 中 | 优化连接参数和功耗 |
| **iOS/Android BLE 兼容性** | 中 | 高 | 多设备测试验证 |
| **BLE 功耗过高** | 中 | 中 | 优化连接间隔和睡眠模式 |
| **GATT 服务发现失败** | 低 | 高 | 使用标准 FIDO GATT UUID |

### 9.3 双模切换风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **USB/BLE 同时连接冲突** | 低 | 中 | 设计优先级策略（USB > BLE） |
| **双模功耗过高** | 中 | 中 | 优化 BLE 功耗，USB 为主 |
| **双模切换延迟** | 低 | 低 | 无缝切换，状态同步 |

### 9.4 协议实现复杂度风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **CTAP over HID 实现复杂** | 中 | 中 | 参考 SoloKeys/OpenSK 实现 |
| **CTAP over BLE 实现复杂** | 中 | 中 | 参考 OpenSK BLE 实现 |
| **传输绑定协议错误** | 低 | 高 | 参考 FIDO2 CTAP 规范 |
| **开发时间延长** | 中 | 中 | 分阶段验证，迭代开发 |

### 9.5 总体风险评估

| 风险等级 | 风险项数 | 说明 |
|----------|----------|------|
| **高** | 2 | USB/BLE 兼容性风险，需多平台测试 |
| **中** | 5 | BLE 稳定性、功耗、协议实现复杂度 |
| **低** | 3 | USB 连接稳定性、HID 描述符、切换延迟 |

**风险缓解策略**：
1. 多平台兼容性测试（Windows/Mac/Linux/iOS/Android）
2. 分阶段验证（USB HID 验证 → BLE GATT 验证 → 双模验证）
3. 参考 SoloKeys/OpenSK 成熟实现
4. ESP-IDF 官方示例验证驱动稳定性

## 十一、替代方案分析

### 11.1 替代方案一：USB only 方案

**方案描述**：
- 通信模式：仅 USB HID
- 芯片：ESP32-S3 内置 USB
- 成本：$0.5-1（仅 USB-C 连接器）

**对比分析**：

| 项目 | USB only | USB + BLE |
|------|----------|-----------|
| **成本** | $0.5-1（更低） | $0.5-1（相同） |
| **便携性** | 差（无无线） | 好（BLE 支持） |
| **兼容性** | 高（USB 稳定） | 中（双模兼容） |
| **功耗** | 低（仅 USB） | 中（BLE 功耗） |
| **适用场景** | 固定桌面 | 通用场景 |

**未选择原因**：
1. 缺少无线便携性，不适合移动场景
2. FIDO2 认证器主流趋势是双模支持
3. BLE 是 FIDO2 推荐的便携认证方式

### 11.2 替代方案二：BLE only 方案

**方案描述**：
- 通信模式：仅 BLE GATT
- 芯片：ESP32-S3 内置 BLE
- 成本：$0（无连接器）

**对比分析**：

| 项目 | BLE only | USB + BLE |
|------|----------|-----------|
| **成本** | $0（最低） | $0.5-1 |
| **桌面支持** | 差（需蓝牙） | 好（USB 直接） |
| **便携性** | 好（无线） | 好（BLE 支持） |
| **功耗** | 低（仅 BLE） | 中（双模功耗） |
| **兼容性** | 中（依赖蓝牙） | 高（双模兼容） |

**未选择原因**：
1. 缺少 USB 支持，不适合桌面场景
2. 桌面环境 BLE 支持不稳定（部分电脑无蓝牙）
3. USB 是 FIDO2 主流桌面认证方式

### 11.3 替代方案三：USB + NFC + BLE 方案

**方案描述**：
- 通信模式：USB + NFC + BLE 全协议
- 芯片：ESP32-S3 + PN532 NFC 模块
- 成本：$5-10（增加 NFC 模块）

**对比分析**：

| 项目 | USB + NFC + BLE | USB + BLE |
|------|-----------------|-----------|
| **成本** | $5-10（更高） | $0.5-1（更低） |
| **协议支持** | 全协议支持 | 双模支持 |
| **适用场景** | 企业级认证器 | 通用认证器 |
| **复杂度** | 高（三协议） | 中（双协议） |
| **功耗** | 高（三模块） | 中（双模块） |

**未选择原因**：
1. 成本显著增加（$5-10），超出项目目标
2. NFC 模块（PN532）增加 PCB 复杂度
3. 项目定位为 DIY 认证器，非企业级
4. 开发复杂度显著增加

### 11.4 替代方案四：外接 USB 芯片 + BLE 模块方案

**方案描述**：
- USB：外接 CH340G/CP2102
- BLE：外接 Nina B112 模块
- 成本：$6-14（外接芯片成本）

**对比分析**：

| 项目 | 外接芯片方案 | 内置方案 |
|------|--------------|----------|
| **成本** | $6-14（显著更高） | $0.5-1（最低） |
| **USB HID 支持** | 需转换芯片 | 内置 HID 设备 |
| **BLE 版本** | BLE 5.0（模块） | BLE 5.0（内置） |
| **PCB 复杂度** | 高（外接芯片） | 低（内置） |
| **稳定性** | 高（独立芯片） | 中（ESP32-S3） |

**未选择原因**：
1. 成本显著增加（$6-14），不符合成本目标
2. UART 转 USB 芯片不支持 HID 设备模式
3. 外接 BLE 模块增加 PCB 复杂度
4. ESP32-S3 内置方案已验证可行
